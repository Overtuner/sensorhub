Initialisierung eines Nodes:


Vorraussetzung:

Auf dem Node ist die Software eingespielt.
Der Node kennt seine Netzwerkadresse (Software!) und ist in Funkreichweite zu seinem übergeordneten Node.

Ablauf der Initialisierung:

Der Node sendet ca. 2 mal je Sekunde eine Nachricht auf Channel 119 mit Value 0. Die Häufigkeit richtet sich dabei nach der Konfiguration des Netzwerkes.
Bei mir gilt folgende Konfiguration je Schleife für die Nodes: 1 Minute schlafen => Radio einschalten => 1 Sekunde schlafen => Radio auswerten und verarbeiten => Radio ausschalten
Kommt diese Nachricht über das Netzwerk beim Server an beginnt dieser mit der Initialisierung.
Die Initialisierung besteht (theoretisch) aus beliebig vielen Nachrichten. Die Umsetzung dieser Nachrichten muss im Setup der Nodesoftware programmiert werden!
Minimum sind folgende Nachrichten:
Channel 111, Value: Länge der Sleeptime in Sekunden
Channel 112, Value: Radiomode in der Sleeptime ( 0 = Radio aus; 1 = Radio ein )
Channel 118, Value: 1 = Init beendet

Der Empfang der Nachricht wird vom Node durch die Rücksendung der Nachricht auf gleichem Channel mit gleichem Value beantwortet.

Die Initialisierung wird einmalig beim Start eines Nodes durchlaufen. Die einzelnen Parameter (Sleeptime, Radiomode, ...) können im laufenden Betrieb durch den Server verändert werden.
